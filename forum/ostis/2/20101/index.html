Разработка редактора исходных текстов баз знаний (KBE) Сообщение DenisKoronchik » 16 апр 2011, 23:40 Изображение kbe_logo_020.png Добрый день всем. С сегодняшнего дня весь процесс разработки редактора инструментальных средств баз знаний (KBE - Knowledge Base source Editor) будет отражаться в данном разделе форума. Процесс обсуждения необходимых возможностей и направления развития инструмента будет происходить здесь. Это дает возможность каждому принять в этом процессе участие. На текущий момент последней версией доступной для скачивания является версия 0.2.0 (http://www.ostis.net/news/reliz-knowledge-base-source-editor-versiya-0.2.0.html). В настоящее время вы ведем разработку версии 0.3.0. Вот лишь небольшой список направлений, по которым производится развитие инструментального средства: поддержка проектов баз знаний - речь идет о том, что инструмент будет создавать проекты на основании директории с исходными текстами и позволять редактировать их различными способами. Файл описания проекта будет содержать все ключевые узлы и другую мета-информацию о проекте, которая позволит работать на уровне не только отдельных фрагментов БЗ, а также и на уровне всей БЗ (К примеру, переименование идентификатора ключевого узла по всему проекту) поддержка RDF/OWL для экспорта/импорта поддержка расширений - на текущий момент реализовано. Редактор SCg-конструкций реализован как дополнение. автоматическое размещение SCg-конструкций - в планах сделать систему автоматического размещения SCg-конструкций, что необходимо для импорта в SCg из других форматов. добавить ограничители в SCg-конструкции - планируем добавить различного рода ограничители. Примером их может быть ограничитель, который будет устанавливать некоторые объекты на одной линии по горизонтали/вертикали. Изменения происходят даже при перемещении объектов. Это значительно может сократить время при редактировании SCg-конструкций. и т. д. Полный список можно посмотреть тут (http://sourceforge.net/apps/trac/ostis/query?group=status&milestone=KBE+0.3.0). Предлагаем активно включиться в обсуждение, это позволит нам быстрее и более качественно осуществлять работу по развитию данного инструмента. P.S.: любое мнение будет учтено. Всем спасибо. DenisKoronchik Сообщение DenisKoronchik » 03 июн 2011, 12:32 На текущий момент работа над редактором продолжается, в настоящее время мы работаем над интерфейсами, которые войдут в SDK и позволят расширить редактор для ваших нужд. Редактор будет позволять создавать редакторы на любых внешних языках, добавлять поддержку новых содержимых в SCg, создавать и улучшать алгоритмы размещения и многое другое.
