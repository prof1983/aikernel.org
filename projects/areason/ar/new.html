Руководство по созданию модуля AR

Модуль AR - программа, подключаемая к ядру (ядрам) AR.

  • Методы подключения

Подключение

Все подключения выполняются в виде класса. Родительский класс TARConnection. Ядро всегда рассматривается как сервер подключений.

Функции работы с подключением

  • Create
  • ConfigureLoad
  • Initialize
  • Finalize
  • ConfigureSave
  • Free

Дополнительные функции работы с классом TARConnectionClient

  • SendToCore - послать команду ядру
  • ReseiveFromCore - Принять команду от ядра

События

  • OnAfterSend - Выполняется после отправки сообщения
  • OnBeforeSend - Выполняется перед отправкой сообщения
  • OnAfterReseive - Выполняется после приема сообщения
  • OnBeforeReseive - Выполняется перед приемом сообщения
  • OnReseive - Пришли данные от ядра

Дополнительные функции работы с классом TARConnectionServer

  • SendFromCore - Послать команду от ядра
  • ReceiveToCore - Принять команду от клиента

События

  • OnAfterSend - Выполняется после отправки сообщения
  • OnBeforeSend - Выполняется перед отправкой сообщения
  • OnAfterReseive - Выполняется после приема сообщения
  • OnBeforeReseive - Выполняется перед приемом сообщения
  • OnReseive - Пришли данные от ядра

Методы подключения

  • Unit - подключение при компиляции (TARConnectionUnit)
  • Socket - Терез TCP/IP соединение (TARConnectionTcpIPClient, TARConnectionTcpIPServer)
  • Plugin - в виде DLL плугина (TARConnectionPluginClient, TARConnectionPluginServer)

Unit - Подключение при компиляции

Модуль выполняются в виде класса. Родительский класс TARModule.

Функции создания и удаления модуля.

  1. Create - Создание объекта. При создании указывается Node конфигураций (AConfig) и Node записи лога выполнения (ALog). AConfig и ANode имеют структуру XML.
  2. ConfigureLoad - загрузка конфигураций.
  3. Initialize - Инициализация объекта.
  4. Finalize - Завершение работы объекта.
  5. ConfigureSave - Сохранение конфигураций в AConfig.
  6. Free - Удаление объекта. Освобождение памяти.

Функции подключения. Для подключения к ядру

Socket - Терез TCP/IP соединение

Модули подключаются через Socket. Вызавается клиентом. Указывается Host и Port сервера ядра AR. Для подключения через Socket созданы 2 класса: TARConnectionTcpIPClient и TARConnectionTcpIPServer.

Для соединения/Разъединеия используйте функции Connect/Disconnect или Initialize/Finalize.

Plugin - в виде DLL плугина

Вызывается сервером (по команде ядро подключается к dll). Указывается имя dll файла и его расположение (путь).

Функции, которые должны быть реализованы в dll.

  • AR_Plugin_Info - Возвращает информацию о плугине, а так же необходимую версию ядра.
  • AR_Plugin_Load - Загрузка модуля - создание соединение.
  • AR_Plugin_UnLoad - Выгрузка модуля - завершение соединения.

Соединение ядер между собой

Соединение ядер происходит через какой либо модуль. Модуль является ретранслятором между двумя ядрами. При этом команды посланые ядром №1 будут посылаться модулю и приниматься ядром №2 как команды от модуля. И наоборот.

Agent Applications

Agent Applications

  • P-Mail - A Private EMail System
    P-Mail is an email and instant messaging system that uses software agents to implement the mail client. The system is designed to ensure that messages remain private. Privacy is assured because messages never reside on any server device. P-Mail is a peer-to-peer messaging service. The message moves from the sending machine agent to the receiving machine agent and is never stored on any intermediate machine. P-Mail is a good example of a peer-to-peer processing application that has significant advantages over the client-server approach. This is also a good example of agents created using the AgentBuilder toolkit
  • Intelligent Agents for Buying and Selling Electric Power in a Dynamic Auction
    Acronymics, Inc. personnel and Alternative Energy Systems Consultants (AESC) under contract to the Electric Power Research Institute (EPRI) have developed a prototype electronic auction for use in buying and selling electric power. This system uses a network of intelligent software agents and each agent represents a stakeholder in the marketplace. A stakeholder wants to buy or sell electricity. This research has shown how communicating, cooperating agents can implement an auction mechanism that does not require a central authority or auctioneer. Markets form and dissolve based on the needs of the various stakeholders. This system was developed using the AgentBuilder Pro product. A demonstration version of the system is available for download.
  • Smart*DER: Intelligent Agents For Distributed Control and Monitoring of Energy Resources
    Under contract to the California Energy Commission (CEC), Alternative Energy Systems Consultants (AESC) and Acronymics, Inc. personnel have developed intelligent agents that communicate and collaborate to schedule operation of distributed energy resources in the California energy market.
